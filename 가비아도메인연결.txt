[기업 서버 PC - Nginx 및 가비아 도메인 외부 연결 가이드]

본 문서는 사내 내부 고정 IP를 사용하는 서버 PC의 Nginx 서비스를 가비아 도메인과 연결하여 외부에서 접속 가능하게 만드는 단계를 설명합니다.

---

1. 준비 단계: 공인 IP(Public IP) 확인
   - 외부에서 우리 회사 네트워크로 들어오기 위한 통로(IP)를 알아야 합니다.
   - 서버 PC의 브라우저에서 '네이버' 접속 후 "내 아이피" 검색
   - 확인된 IP (예: 211.xxx.xxx.xxx)를 메모해 둡니다.

2. 사내 네트워크 설정 (방화벽 및 포트 포워딩)
   ※ 이 작업은 사내 네트워크 관리자(IT팀)의 협조가 필요할 수 있습니다.
   - 포트 포워딩(Port Forwarding): 외부 공인 IP의 80(HTTP) 포트 및 443(HTTPS) 포트로 들어오는 요청을 서버 PC의 '내부 고정 IP'로 연결하도록 설정합니다.
   - 방화벽 허용: 사내 방화벽 장비에서 외부(Any)로부터 해당 서버 PC IP로의 80, 443 포트 통신을 허용(Allow) 규칙에 추가합니다.

3. 가비아(Gabia) DNS 설정
   - 가비아 홈페이지 로그인 > My가비아 > 도메인 관리 > DNS 설정
   - 해당 도메인의 [설정] 클릭 > [레코드 수정] 클릭
   - 아래와 같이 레코드 추가:
     * 타입: A
     * 호스트: @ (도메인 자체 접속용)
     * 값/위치: 위 1번에서 확인한 '회사의 공인 IP' 입력
     * TTL: 3600 (기본값)
   - 동일하게 호스트 'www'도 추가하면 좋습니다. (www.도메인 접속용)

4. Nginx 설정 파일 수정
   - 서버 PC의 Nginx 설정 파일(보통 nginx.conf 또는 sites-available/default)을 수정합니다.
   - server_name 부분에 실제 도메인을 적어주어야 정확한 라우팅이 가능합니다.

   [설정 예시]
   server {
       listen       80;
       server_name  yourdomain.com www.yourdomain.com; # 가비아에서 산 도메인 입력

       location / {
           root   /path/to/your/project/build; # 빌드된 파일 경로
           index  index.html;
           try_files $uri $uri/ /index.html; # React/Vue 라우팅 대응
       }
   }

5. 서비스 재시작 및 확인
   - Nginx 설정 테스트: nginx -t
   - Nginx 재시작: sudo systemctl restart nginx (또는 brew services restart nginx)
   - 외부 네트워크(모바일 핫스팟 등)에서 도메인 주소로 접속이 되는지 확인합니다.

6. 추가 권장 사항: SSL(HTTPS) 적용
   - 보안과 신뢰성을 위해 HTTPS 적용이 필수적입니다.
   - Certbot(Let's Encrypt)을 설치하여 무료로 SSL 인증서를 발급받고 Nginx에 적용하는 것을 권장합니다.
